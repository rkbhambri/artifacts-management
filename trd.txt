Pit Coding Interview Assignment  Solutions Engineer, Implementation  
Background  
At Pit we turn human intent into reliable software. Our core pipeline takes a product  requirement (PRD), refines it into a specification, has an AI agent write the code, and  deploys the result. The platform is structured so that each customer can have one or  more systems, and within each system the pipeline produces a set of artifacts like  source code, configuration files, migration scripts, and other generated outputs.  
During the pipeline run, various intermediate and final artifacts are produced. These  artifacts need to be stored, versioned, and made available to both internal tooling and the  
customer’s deployment infrastructure. The content is often sensitive — it may contain  proprietary business logic, database schemas, or integration credentials specific to the  customer’s environment.  
Description of Problem  
You are tasked with building the first iteration of a backend API to handle artifact storage for  a pipeline run. The clients to this API are both internal services (the pipeline agents) and  web browsers (the Pit dashboard). The functionality involves:  
1. Uploading an artifact to a specific system. No processing of the file is needed for now.
2. Listing all stored artifacts for a given system. 
3. Notifying connected clients that a new artifact is available for them to retrieve. How clients are notified is up to you. Clients in this case are web browsers.  
Note: You are allowed to use any technical tool of your choosing to complete the task with  the exception that it should be you as an individual that contributes the code. Coding agents  or similar tools are fine to use.  
We understand that you have other commitments and therefore ask you to spend at most  five hours on this assignment, preferably less. We’d rather see that you prioritize effectively  than try to build everything you can think of.  
Technical Requirements  
1. Artifacts should be persisted across service restarts. How this is achieved is up to you. 
2. Include a README describing how to run the application using docker / docker-compose. 
3. The solution should be written in TypeScript. 
4. You are free to make assumptions, but please document them in the README so we can discuss them together. 
If things are unclear with the problem description itself don’t hesitate to reach out.  Evaluation Criteria
